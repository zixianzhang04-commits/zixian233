# 执行步骤与开发计划

## 总原则
- 小步快跑，分阶段验证
- 先完成基础功能，再补充统计、预算、移动端
- 保证数据安全与版本稳定
- 每阶段完成后自测

## 阶段划分

### 阶段 1 — 基础记账 ✅ 已完成
- Electron + React + Vite 骨架
- SQLite / sql.js 离线数据库
- 记账 CRUD（金额/类型/分类/日期/备注）
- 分类管理（增删改、20 个预设分类）
- 基础 UI 布局

### 阶段 2 — 分类进阶 + 撤销/重做 ✅ 已完成
- 子分类树状管理 + 排序
- 撤销/重做（create/update/delete 操作栈）
- 日期范围筛选
- 快速复制上一条

### 阶段 3 — 导入导出 + 备份 ✅ 已完成
- CSV / Excel 导出
- CSV / Excel 导入（预览 + 字段映射 + 校验）
- JSON 完整备份与恢复
- import/export UI 组件

### 阶段 4 — 统计 + 预算 ✅ 已完成
- 月度收支总览卡片
- 支出分类占比饼图（Recharts）
- 近 7/30 天趋势折线图
- 月度预算设置 + 进度条 + 超支警告

### 阶段 5 — UI 设计规范 ✅ 已完成
- 完整 Design Token 系统
- Inter 字体 + 8px 网格 + 缓动动画
- 页面入场 stagger / 悬停反馈 / 波纹效果
- PWA manifest + Service Worker + 图标
- Cloudflare Pages 部署

### 阶段 M1 — 移动端响应式布局（当前）
1. 底部 Tab 导航栏（≤768px 显示，替代侧边栏）
2. 侧边栏桌面保留 / 移动端隐藏
3. 安全区域适配（iPhone notch + home indicator）
4. 视口元标签确认
5. 内容区全宽撑满

### 阶段 M2 — 组件移动端适配
1. 表格 → 卡片式列表（每条记录一张卡片）
2. 统计卡片 → 垂直堆叠
3. 图表 → 减小尺寸 + 自适应
4. 表单 → 全宽输入 + 更大触摸目标（≥44px）
5. Modal → 底部弹出 Sheet 式

### 阶段 M3 — 触摸交互增强
1. 滑动返回 / 侧滑操作
2. 下拉刷新
3. 长按快捷菜单
4. iOS 触觉反馈
5. 手势冲突处理

### 阶段 M4 — PWA 完整体验
1. 启动画面（Splash Screen）
2. 离线状态提示
3. 应用更新提示
4. 后台同步
5. 推送通知

## 里程碑

| 里程碑 | 内容 |
|--------|------|
| M1 | 用户登录 + 基础记账 CRUD ✅ |
| M2 | 分类管理 + 快速复制 + 撤销/重做 ✅ |
| M3 | 导入导出 + 自动备份 ✅ |
| M4 | 统计图表 + 预算提醒 ✅ |
| M5 | UI 完整设计系统 ✅ |
| M6 | 移动端响应式布局 |
| M7 | PWA 完整体验 |

## 风险控制
- 移动端改动渐进式，不影响桌面版功能
- 每次提交保持可回滚状态
- 关键交互（记账 CRUD）优先保证稳定性
- 数据存储层不变，只改 UI 表现层
