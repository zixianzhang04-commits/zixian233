# 执行步骤与开发计划

## 1. 总原则
- 小步快跑，分阶段验证
- 先完成基础功能，再补充统计与预算
- 保证数据安全与版本稳定
- 避免一次性做太多，优先可运行的 MVP

## 2. 阶段划分

### 阶段 1：项目初始化与基础架构
- 初始化 Electron 项目
- 搭建前端框架（React/Vue）
- 设计并初始化本地 SQLite 数据库
- 完成用户登录、注册、多用户切换
- 实现基础记账 CRUD
- 搭建 `CLAUDE.md`、`docs/` 和 `dev-logs/` 目录

### 阶段 2：分类管理与记录交互
- 完成分类管理功能：新增/编辑/删除、子分类、图标、排序
- 实现记账列表筛选、按时间倒序显示
- 添加快速复制上一条记录
- 实现撤销/重做基础功能

### 阶段 3：导入导出与备份
- 实现 Excel/CSV 导入与导出
- 支持导入字段映射与预览
- 完成本地自动备份机制
- 增加备份恢复入口

### 阶段 4：统计与预算
- 实现月度收支总览卡片
- 添加支出占比饼图、趋势折线图、年度柱状图
- 实现预算设置与实时超预算弹窗提醒
- 完成收支预测功能

### 阶段 5：优化与验收
- 优化 UI/UX，确定淡蓝色主题
- 完成多用户隔离与账号保护场景测试
- 修复已知问题，补齐文档
- 进行稳定性测试与数据一致性检查

## 3. 每日开发流程
1. 先阅读 `docs/execution-plan.md` 的当前阶段任务
2. 运行 `node scripts/daily-log.js` 生成当天日志
3. 开发并完成一个小功能点
4. 在当天日志中记录“完成事项”和“待办事项”
5. 若发现需求或设计变更，立即补充至 `docs/requirements.md`

## 4. 里程碑示例
- M1：用户登录 + 基础记账 CRUD
- M2：分类管理 + 快速复制 + 撤销/重做
- M3：Excel 导入导出 + 自动备份
- M4：统计图表 + 预算提醒
- M5：整体 UI 完成 + 质量验收

## 5. 风险控制
- 先保证数据库结构稳定，再做复杂业务
- 导入导出功能需严格异常校验
- 预算、分类删除、撤销/重做等关键逻辑需编写详细文档
- 数据备份与恢复要作为基础保障功能实现
